Component / Core Target What Was Asked Of Me What I Have Done
vote & result Web Apps Run them inside a local Kubernetes cluster, ensure they can process and display votes. Modified source files to listen on unprivileged port `8080`. Exposed them properly using services & host Ingress records.
Security Hardening Implement non-root permissions and run containers safely. Updated Dockerfiles to run as custom non-root system users (`appuser` (10001) for vote, `node` (1000) for result) instead of default root access.
Postgres Storage Maintain database state. Vote logs must survive pod terminations and deletion tests. Refactored database to a StatefulSet mapping to local PersistentVolumeClaims (`PVC`). Tested pod restart persistence.
Configs & Secrets Keep DB credentials secure. Do not leave credentials hardcoded in standard manifests. Decoupled credentials and created `db-secret.yaml`. Dynamically injected credential strings via standard `valueFrom` environment mappings.
Ingress Routing Avoid exposing static NodePorts. Route traffic cleanly using an Ingress rule. Configured NGINX Ingress rules using host routing: vote.127.0.0.1.nip.io and result.127.0.0.1.nip.io.
Autoscaling (HPA) Implement a scaling mechanism dynamically managing replication sizes. Created HorizontalPodAutoscaler (`HPA`) rules targetting vote frontend replicas between 1 and 10 based on CPU load thresholds.
Network Policies Configure Network Policies to restrict connection traffic boundaries. Created micro-segmentation network policies. Allowed ingress to frontends from the Ingress controller via `namespaceSelector: {}` while restricting backend data stores (Redis & Postgres) to authorized application pods.
DB Bootstrapping Initialize the PostgreSQL database schema gracefully. Created an independent setup Job hook `db-bootstrap-job.yaml` that verifies postgres status using `pg_isready` before executing schema creation queries.
Probes & Healthchecks Add liveness and readiness checks to prevent routing traffic to dead pods. Mapped robust TCP socket checks (Redis), HTTP query probes (`/` for vote/result web engines), and script process state indicators (Worker).
CI/CD Pipeline Deploy GitHub Action triggers targetting `/vote/**` checking lint rules and image updates. Implemented pipeline workflow file `.github/workflows/ci.yml`. Configured Python linting, Docker multi-stage compiles, Kind spinups, and curl validations.
