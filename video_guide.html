DevOps Presentation Video Guide

A structured step-by-step workflow and script to record your 5-minute Loom walkthrough for the SquareOps DevOps Candidate Take-Home.

Total Time: ~5 Mins Target Tool: Loom / Zoom
1

Introduction & Repository Tour

0:00 - 0:30 (30s)
What to Show

Show your camera and the repository files in your editor (e.g. VS Code).

Suggested Talking Script
"Hello, my name is [Your Name], and this is my submission for the SquareOps DevOps Engineer take-home assignment. Today, I'll walk you through how I migrated the Docker Voting App onto a production-ready Kubernetes setup. Let's start with a quick tour of the repository. Over in the file tree, you'll see I created a standardized Helm Chart under 'charts/voting-app' containing all templates and parameters. I've also built automated bootstrap scripts for both Linux and Windows environments, and created a GitHub Actions workflow in '.github/workflows/ci.yml'."
2

Live Demo: Local Verification

0:30 - 2:00 (90s)
What to Show

Open your terminal, show the running pods, then open your browser and showcase the voting flow.

Suggested Talking Script
"I'll showcase the running cluster. Using Kind, our entire stack has been deployed. If we check the pods, all five core services—the vote frontend, redis queue, .NET worker, postgres database, and result dashboard—are running healthily. Since we're using NGINX Ingress, I can navigate to 'vote.127.0.0.1.nip.io' and cast a vote for Cats. When I check the results dashboard at 'result.127.0.0.1.nip.io', we can see the vote tally updates in real time. We can also tail the logs of the C# worker pod to see it processing the queue entries..."
Useful Commands to Copy
kubectl get pods
kubectl get ingress
kubectl logs -l app=worker --tail=10
3

Helm Chart & Core Architecture Decision

2:00 - 3:30 (90s)
What to Show

Open `charts/voting-app/templates/db-statefulset.yaml` and `networkpolicies.yaml` in your editor.

Suggested Talking Script
"Next, let's look at the Helm Chart templates. One key decision I made was refactoring the Postgres database from a plain Deployment to a StatefulSet with a PersistentVolumeClaim. This ensures vote data persists reliably across container crashes or manual deletions. I also decoupled the administrative passwords using Secrets. In terms of security, I implemented Network Policies. Ingress is restricted so that only the NGINX controller can talk to the frontends, and the database and Redis instances are blocked from any direct public access, allowing connections only from authorized consumer pods."
4

CI/CD Pipeline Demonstration

3:30 - 4:30 (60s)
What to Show

Open your web browser and show a successful build run in the GitHub Actions 'Actions' tab.

Suggested Talking Script
"For the CI/CD pipeline, I configured a GitHub Actions workflow that triggers on code pushes under '/vote/**'. The pipeline lints both the Python codebase and the Helm templates to catch format issues. Next, it compiles the image and publishes it to GitHub Container Registry. Finally, it provisions a temporary kind cluster in the pipeline runner, deploys the Helm chart, and runs an automated curl smoke-test validation checking for HTTP 200 OK."
5

Production Trade-offs & Improvements

4:30 - 5:00 (30s)
What to Show

Show your camera or return to the README.md trade-offs section.

Suggested Talking Script
"Given more time for a production environment, I would make a few changes. First, I'd replace the local hostPath storage with an enterprise CSI driver like AWS EBS. Second, I'd set up cert-manager to manage SSL/TLS certificates automatically. Finally, I'd move secret storage to a cloud KMS provider like AWS Secrets Manager or HashiCorp Vault. Thank you for your time, and I look forward to our follow-up discussion!"

Recording Best Practices

  • Cluster Status: Make sure your kind cluster is running and healthy before clicking record.
  • Resolution Check: Zoom in slightly in VS Code (Ctrl + or Cmd +) so the text is highly readable in the video.
  • Clean Environment: Close extra browser tabs and chat systems to avoid notifications during recording.
  • Audio Quality: Use a clear microphone and record in a quiet room.
